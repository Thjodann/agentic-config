# `.ai/` - Canonical Agentic Config

This folder is the shared source of truth for repo-scoped agentic behavior. You can
author here directly, or create native assets in Cursor, Windsurf/Devin, Claude Code,
Codex, or Continue and then adopt them into `.ai/`.

The generator compiles canonical assets into the layout each supported IDE expects so
every developer gets comparable rules, commands, skills, and agents no matter which
agentic IDE they use.

## Golden Rules

- After adoption, edit `.ai/` as the durable source of truth.
- Generated files contain `AUTOGENERATED` markers and are overwritten by sync.
- Native files without an autogen marker are treated as contribution candidates.
- Obsidian is optional: these are normal Markdown files with frontmatter.

## Layout

```
.ai/
├── INDEX.md
├── README.md
├── sync.py
├── .manifest.json
├── rules/<name>.md
├── agents/<name>.md
├── commands/<name>.md
└── skills/<name>/
    └── SKILL.md
```

## Commands

```bash
./sync-agentic.sh
./sync-agentic.sh sync
./sync-agentic.sh --check
./sync-agentic.sh check
./sync-agentic.sh doctor
./sync-agentic.sh doctor --staged
./sync-agentic.sh adopt cursor .cursor/rules/my-rule.mdc
./sync-agentic.sh adopt windsurf .windsurf/workflows/release.md
./sync-agentic.sh adopt codex .agents/skills/debug/SKILL.md
./sync-agentic.sh adopt claude .claude/agents/reviewer.md
./sync-agentic.sh adopt --all
./sync-agentic.sh reconcile skill debug-word
./sync-agentic.sh reconcile --all-exact
./sync-agentic.sh clean
./sync-agentic.sh clean --native-duplicates
./sync-agentic.sh bootstrap
```

`doctor` reports native-only assets, stale generated files, canonical conflicts, and
known degradations. `adopt` promotes native IDE files into `.ai/`; `reconcile`
canonicalizes exact duplicate native assets; `clean` removes generated local
projections; `bootstrap` generates projections after clone.

## Canonical Frontmatter

All assets require `description`. `name` is optional and defaults to the file or
directory name.

```yaml
---
name: <slug>
description: Explain exactly when this applies.
targets: claude, cursor
---
```

Rules:

```yaml
---
name: api-style
description: API conventions for handlers and service boundaries.
activation: model_decision # always_on | model_decision | glob | manual
globs: app/api/**/*.py
---
```

Agents:

```yaml
---
name: code-reviewer
description: Reviews diffs for correctness, regressions, and missing tests.
tools: Read, Grep, Glob
model: sonnet
---
```

Commands:

```yaml
---
name: summarize-changes
description: Summarizes the current diff and risk areas.
argument-hint: "[base-branch]"
---
```

Skills:

```yaml
---
name: debug-word-addin
description: Debugs Word add-in startup, Docker health, and local HTTPS issues.
allowed-tools: Bash, Read
---
```

## Mapping

| Canonical | Claude Code | Cursor | Windsurf/Devin | Codex | Continue |
| --- | --- | --- | --- | --- | --- |
| rule | `.claude/rules/*.md` | `.cursor/rules/*.mdc` | `.devin/rules/*.md` | managed `AGENTS.md`, plus conditional `rule-*` skills | `rule-*` prompts |
| command | `.claude/commands/*.md` and `.claude/skills/*` | `.cursor/commands/*.md` | `.windsurf/workflows/*.md` | explicit `command-*` skills shown as `⚡ /name` | prompts |
| skill | `.claude/skills/*/SKILL.md` | `.cursor/skills/*/SKILL.md` | `.windsurf/skills/*/SKILL.md` | `.agents/skills/*/SKILL.md` | prompts |
| agent | `.claude/agents/*.md` | `.cursor/agents/*.md` | degraded `agent-*` skills | `.codex/agents/*.toml` | `agent-*` prompts |

Codex `.codex/rules/*.rules` files are execution policy, not portable behavioral
guidance. `doctor` flags them as Codex-only instead of adopting them as team rules.

## Native Contribution Workflow

1. Create the asset in your IDE.
2. Run `./sync-agentic.sh doctor`.
3. Adopt the native file into `.ai/`.
4. Run `./sync-agentic.sh`.
5. Commit `.ai/`, `.ai/.manifest.json`, root `AGENTS.md`, and toolkit files. In
   source-only mode, leave generated IDE folders ignored.

Examples:

```bash
./sync-agentic.sh adopt cursor .cursor/rules/service-style.mdc
./sync-agentic.sh adopt windsurf .windsurf/workflows/release-check.md
./sync-agentic.sh adopt codex .codex/agents/reviewer.toml
./sync-agentic.sh
```

## Source-Only Workflow

Generated IDE folders are local projections and may be ignored by Git. Commit `.ai/`,
`sync-agentic.sh`, docs/tests, `.ai/.manifest.json`, and root `AGENTS.md`; each
developer can run this after cloning:

```bash
./sync-agentic.sh bootstrap
```

If several teammates created the same native skill/rule/command/agent before adopting
it, use:

```bash
./sync-agentic.sh doctor
./sync-agentic.sh reconcile --all-exact
./sync-agentic.sh
```

`doctor` never changes files. `reconcile` only auto-merges exact normalized duplicates.
Same-name assets with different content are reported for manual resolution.

## Safe Deletes

The v2 manifest records generated files, hashes, and managed blocks. Removing a
canonical asset from `.ai/` and rerunning sync prunes only files or managed blocks that
still look like this tool's output. Markerless human files at generated paths are
reported instead of overwritten or deleted.

`./sync-agentic.sh clean` removes generated IDE projection files listed in the manifest
while keeping `.ai/` and root `AGENTS.md`. It refuses markerless files whose bytes no
longer match the manifest hash. Add `--native-duplicates` only when you want to remove
unmanaged native files that exactly match canonical assets.
