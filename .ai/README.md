# `.ai/` - Canonical Agentic Config

This folder is the shared source of truth for repo-scoped agentic behavior. You can
author here directly, or create native assets in Cursor, Windsurf/Devin, Claude Code,
Codex, or Continue and then adopt them into `.ai/`.

The generator compiles canonical assets into the layout each supported IDE expects so
every developer gets comparable rules, commands, skills, and agents no matter which
agentic IDE they use.

## Golden Rules

- After adoption, edit `.ai/` as the durable source of truth.
- Generated files contain `AUTOGENERATED` markers and are overwritten by sync.
- Native files without an autogen marker are treated as contribution candidates.
- Obsidian is optional: these are normal Markdown files with frontmatter.

## Layout

```
.ai/
├── INDEX.md
├── README.md
├── sync.py
├── .manifest.json
├── rules/<name>.md
├── agents/<name>.md
├── commands/<name>.md
└── skills/<name>/
    └── SKILL.md
```

## Commands

```bash
agentic-config sync
agentic-config check
agentic-config doctor
agentic-config bootstrap
agentic-config adopt cursor .cursor/rules/my-rule.mdc
agentic-config adopt --all
agentic-config reconcile --all-exact
agentic-config demote cursor .cursor/commands/my-command.md
agentic-config promote cursor .cursor/commands/my-command.md
agentic-config clean
agentic-config clean --native-duplicates

./sync-agentic.sh
./sync-agentic.sh sync
./sync-agentic.sh --check
./sync-agentic.sh check
./sync-agentic.sh doctor
./sync-agentic.sh doctor --staged
./sync-agentic.sh adopt cursor .cursor/rules/my-rule.mdc
./sync-agentic.sh adopt windsurf .windsurf/workflows/release.md
./sync-agentic.sh adopt codex .agents/skills/debug/SKILL.md
./sync-agentic.sh adopt claude .claude/agents/reviewer.md
./sync-agentic.sh adopt --all
./sync-agentic.sh reconcile skill debug-word
./sync-agentic.sh reconcile --all-exact
./sync-agentic.sh demote cursor .cursor/commands/debug-word.md
./sync-agentic.sh promote cursor .cursor/commands/debug-word.md
./sync-agentic.sh clean
./sync-agentic.sh clean --native-duplicates
./sync-agentic.sh bootstrap
```

`agentic-config` is the preferred interface. `./sync-agentic.sh` remains the
repo-local compatibility wrapper. `doctor` reports native-only assets, stale
generated files, canonical conflicts, and known degradations. `adopt` promotes
native IDE files into `.ai/`; `reconcile` canonicalizes exact duplicate native
assets; `demote` suppresses one generated path for one consumer IDE; `promote`
restores that path; `clean` removes generated local projections; `bootstrap`
generates projections after clone. `doctor` also checks supported user-level
directories like `~/.cursor/` and `~/.codex/` for assets that overlap this repo's
`.ai/` source, while ignoring unrelated global-only personal assets. Global
overlaps are reported as user-level warnings and do not fail `doctor` or
`bootstrap` on their own. If a repo-native asset matches a global asset, the
global asset takes priority and the repo-native asset is not promoted into `.ai/`.

The compiler can also run from an external canonical source by setting
`AGENTIC_CONFIG_REPO_ROOT` and `AGENTIC_CONFIG_AI_DIR`. The CLI uses this for
stealth mode, where local source lives in `.agentic-config/.ai/` while generated
IDE projections still land in the repo root.

## Canonical Frontmatter

All assets require `description`. `name` is optional and defaults to the file or
directory name.
Use behavior names rather than origin-IDE names: prefer `runbook-naming` over
`cursor-runbook-naming`. Generated projections strip leading IDE prefixes, and
`doctor` reports canonical sources that still need to be renamed.

```yaml
---
name: <slug>
description: Explain exactly when this applies.
targets: claude, cursor
---
```

Rules:

```yaml
---
name: api-style
description: API conventions for handlers and service boundaries.
activation: model_decision # always_on | model_decision | glob | manual
globs: app/api/**/*.py
---
```

Agents:

```yaml
---
name: code-reviewer
description: Reviews diffs for correctness, regressions, and missing tests.
tools: Read, Grep, Glob
model: sonnet
---
```

Commands:

```yaml
---
name: summarize-changes
description: Summarizes the current diff and risk areas.
argument-hint: "[base-branch]"
---
```

Skills:

```yaml
---
name: debug-word-addin
description: Debugs Word add-in startup, Docker health, and local HTTPS issues.
allowed-tools: Bash, Read
---
```

## Mapping

| Canonical | Claude Code | Cursor | Windsurf/Devin | Codex | Continue |
| --- | --- | --- | --- | --- | --- |
| rule | `.claude/rules/*.md` | `.cursor/rules/*.mdc` | `.devin/rules/*.md` | managed `AGENTS.md`, plus conditional `rule-*` skills | `rule-*` prompts |
| command | `.claude/commands/*.md` and `.claude/skills/*` | `.cursor/commands/*.md` | `.windsurf/workflows/*.md` | explicit `command-*` skills shown as `⚡ /name` | prompts |
| skill | `.claude/skills/*/SKILL.md` | `.cursor/skills/*/SKILL.md` | `.windsurf/skills/*/SKILL.md` | `.agents/skills/*/SKILL.md` | prompts |
| agent | `.claude/agents/*.md` | `.cursor/agents/*.md` | degraded `agent-*` skills | `.codex/agents/*.toml` | `agent-*` prompts |

Codex `.codex/rules/*.rules` files are execution policy, not portable behavioral
guidance. `doctor` flags them as Codex-only instead of adopting them as team rules.

## Native Contribution Workflow

1. Create the asset in your IDE.
2. Run `agentic-config doctor`.
3. Adopt the native file into `.ai/`.
4. Run `agentic-config sync`.
5. Commit `.ai/`, `.ai/.manifest.json`, root `AGENTS.md`, and toolkit files. In
   source-only mode, leave generated IDE folders ignored.

If `doctor` reports that a repo-native asset is shadowed by a global asset, leave
the repo copy unpromoted. Global assets stay user-level and are not imported into
the repo source.

Examples:

```bash
agentic-config adopt cursor .cursor/rules/service-style.mdc
agentic-config adopt windsurf .windsurf/workflows/release-check.md
agentic-config adopt codex .codex/agents/reviewer.toml
agentic-config sync
```

## Source-Only Workflow

Generated IDE folders are local projections and may be ignored by Git. Commit `.ai/`,
`sync-agentic.sh`, docs/tests, `.ai/.manifest.json`, and root `AGENTS.md`; each
developer can run this after cloning:

```bash
agentic-config bootstrap
```

If several teammates created the same native skill/rule/command/agent before adopting
it, use:

```bash
agentic-config doctor
agentic-config reconcile --all-exact
agentic-config sync
```

`doctor` never changes files. `reconcile` only auto-merges exact normalized duplicates.
Same-name assets with different content are reported for manual resolution.
Repo-native assets that match global assets by exact fingerprint, portable
type/name, or normalized body are skipped during adoption so the same callable
behavior does not appear twice in one repo.

If Cursor surfaces multiple generated projections for the same concept, use an
explicit demotion:

```bash
agentic-config doctor
agentic-config demote cursor .cursor/commands/debug-word.md
agentic-config check
```

Demotions are stored in `.ai/.demotions.json`, or `.agentic-config/.ai/.demotions.json`
for stealth installs. They suppress generated output only; markerless native files
are not deleted or archived. Demoting `.agents/skills/command-*` paths suppresses a
Codex-target projection too, so `doctor` prints a warning before suggesting it.

## Safe Deletes

The v2 manifest records generated files, hashes, and managed blocks. Removing a
canonical asset from `.ai/` and rerunning sync prunes only files or managed blocks that
still look like this tool's output. Markerless human files at generated paths are
reported instead of overwritten or deleted.

`agentic-config clean` removes generated IDE projection files listed in the manifest
while keeping `.ai/` and root `AGENTS.md`. It refuses markerless files whose bytes no
longer match the manifest hash. Add `--native-duplicates` only when you want to remove
unmanaged repo-native files that exactly match canonical assets. Global user-level
assets are read-only: this kit reports overlaps but does not promote or clean them.
Use `demote <ide> <generated-path>` only for generated paths reported by `doctor`;
use `promote <ide> <generated-path>` to restore them.
