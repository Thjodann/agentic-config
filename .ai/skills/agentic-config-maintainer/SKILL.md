# Agentic Config Maintainer

Act as the operator for this repo's shared `.ai/` config. The user should be able to
ask in plain language; infer the correct command sequence and handle the arguments.

## Start

1. Run `./sync-agentic.sh doctor` unless the user only asks a conceptual question.
2. Read the relevant `.ai/` asset or native IDE file before changing it.
3. Decide the safest action from the user's intent and the doctor output.

## Actions

- Add a new shared asset: create the canonical `.ai/rules`, `.ai/commands`,
  `.ai/skills`, or `.ai/agents` file directly, then run sync and check.
- Adopt one native asset: infer the IDE from the path and run
  `./sync-agentic.sh adopt <ide> <path>`.
- Reconcile duplicates: run `./sync-agentic.sh reconcile <kind> <name>` for one
  exact group, or `./sync-agentic.sh reconcile --all-exact` when doctor reports
  exact duplicate groups and the user asks to reconcile all of them.
- Bootstrap a clone: run `./sync-agentic.sh bootstrap`.
- Clean generated files: run `./sync-agentic.sh clean` only when the user asks to
  remove local generated projections.
- Clean native duplicates: run `./sync-agentic.sh clean --native-duplicates` only
  when doctor reports exact native duplicates already represented in `.ai/`.

After any mutation, run `./sync-agentic.sh` and `./sync-agentic.sh --check`.

## Asking

Ask only when the next action is ambiguous, such as:

- the user wants a new asset but did not say rule, command, skill, or agent;
- doctor reports same-name assets with different content;
- multiple native-only assets could match the user's request;
- cleanup would remove unmanaged native files.

## Guardrails

- Do not expect the user to know `sync-agentic.sh` arguments.
- Do not treat Codex `.codex/rules/*.rules` files as portable behavioral rules; they are execution policy.
- Do not manually edit generated files with an `AUTOGENERATED` marker.
- Prefer canonical `.ai/` edits over native IDE edits after adoption.
- Report conflicts plainly and stop before choosing a winner unless the user gave enough direction.
