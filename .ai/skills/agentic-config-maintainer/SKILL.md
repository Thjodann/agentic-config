# Agentic Config Maintainer

Act as the operator for this repo's shared `.ai/` config. The user should be able to
ask in plain language; infer the correct command sequence and handle the arguments.

## Start

1. Run `agentic status` unless the user only asks a conceptual question.
   If the global CLI is unavailable but `./sync-agentic.sh` exists, use the wrapper.
2. Read the relevant `.ai/` asset or native IDE file before changing it.
3. Decide the safest action from the user's intent and the status output.

## Actions

- Add a new shared asset: create the canonical `.ai/rules`, `.ai/commands`,
  `.ai/skills`, or `.ai/agents` file directly, then run sync and status.
- Import one native asset: run `agentic import <path>`.
- Import duplicates: run `agentic import --exact-duplicates` when status reports
  exact duplicate groups and the user asks to reconcile all of them.
- Bootstrap or set up a clone: run `agentic setup <repo>`.
- Stealth setup: run `agentic setup --stealth <repo>` only when the user
  explicitly wants local ignored projections without tracked Git changes.
- Clean generated files: run `agentic clean` only when the user asks to
  remove local generated projections.
- Clean native duplicates: run `agentic clean --native-duplicates` only
  when status reports exact repo-native duplicates already represented in `.ai/`.
- Clean global native duplicates: run
  `agentic clean --native-duplicates --global` only when status reports
  exact user-level duplicates and the user explicitly wants global cleanup.

After any mutation, run `agentic sync` and `agentic status`.

## Asking

Ask only when the next action is ambiguous, such as:

- the user wants a new asset but did not say rule, command, skill, or agent;
- status reports same-name assets with different content;
- multiple native-only assets could match the user's request;
- cleanup would remove unmanaged native files, especially from user-level
  directories such as `~/.cursor/` or `~/.codex/`.

## Guardrails

- Do not expect the user to know `agentic` or `sync-agentic.sh` arguments.
- In stealth mode, do not edit `.gitignore`; tracked generated paths and tracked
  `AGENTS.md` must be skipped and reported.
- Do not treat Codex `.codex/rules/*.rules` files as portable behavioral rules; they are execution policy.
- Do not manually edit generated files with an `AUTOGENERATED` marker.
- Prefer canonical `.ai/` edits over native IDE edits after import.
- Report conflicts plainly and stop before choosing a winner unless the user gave enough direction.
- Prefer the intent-level commands (`setup`, `status`, `import`, `sync`, `clean`)
  over advanced implementation commands when they fit.
